üîß V9.43 - Modal Statistics Fix

Verification of Complete Player Statistics in All Modals

‚ùå Problem Identified

The player statistics modals were not consistently showing ALL data when opened from different pages:

  • From Riepilogo Convocazioni: Player modal showed convocazioni stats but training stats were inconsistent
  • From Allenamenti page: Training modal showed training stats but convocazioni stats were inconsistent
  • Root cause: Nested if-blocks prevented proper data population

üêõ Technical Issue (V9.42)

Before (Problematic Code):

if (modalTrainingStatsInPlayer) { const trainingData = getPlayerTrainingData(playerName); // Populate stats... if (modalTrainingTrendInPlayer) { // This only runs if BOTH conditions are true! modalTrainingTrendInPlayer.innerHTML = ...; } if (modalTrainingChartInPlayer) { // This only runs if first condition is true! modalTrainingChartInPlayer.innerHTML = ...; } }

Issue: The nested structure meant that if modalTrainingStatsInPlayer was null or missing, NONE of the other training elements (trend, chart, sessions) would be populated, even if they existed.

‚úÖ Solution (V9.43)

After (Fixed Code):

// Get training data once for all elements const trainingData = getPlayerTrainingData(playerName); if (modalTrainingStatsInPlayer) { modalTrainingStatsInPlayer.innerHTML = ...; } if (modalTrainingTrendInPlayer) { // Independent check - runs regardless of other elements modalTrainingTrendInPlayer.innerHTML = ...; } if (modalTrainingChartInPlayer) { // Independent check - runs regardless of other elements modalTrainingChartInPlayer.innerHTML = ...; } if (modalTrainingSessionsInPlayer) { // Independent check - runs regardless of other elements modalTrainingSessionsInPlayer.innerHTML = ...; }

Fix: Each modal element is now checked independently. Data is fetched once, then each element is populated only if it exists. This ensures maximum compatibility.

üìã Changes Summary

Modified Function:

openPlayerModal(playerName, trend, chartData, recentCallups)

Key Changes:

  • Moved getPlayerTrainingData() call outside the first if-block
  • Made all modal element checks independent (no nesting)
  • Ensured each element is populated if it exists
  • Training data is fetched once and reused for all elements
